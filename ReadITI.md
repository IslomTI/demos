# 🏆 Демоэкзамен 2025: Модуль 1 (Сетевая инфраструктура)

Данный репозиторий содержит автоматизированный Bash-скрипт для мгновенного развертывания Модуля 1 по стандартам чемпионатов "Профессионалы". Скрипт настраивает базовую маршрутизацию, VLAN, NAT, GRE-туннель, OSPF, DHCP, DNS (Прямую и обратные зоны) и безопасность SSH.

---

## ⚠️ ВАЖНО: Подготовка сети в VMware (Ручной этап)
Скрипт не сможет работать, если "виртуальные провода" подключены неверно. До запуска машин настройте сетевые адаптеры (Network Adapters) строго по этой схеме:

### 🌐 Провайдер (ISP) - 3 адаптера:
* **Adapter 1:** `NAT` (Обеспечивает выход в реальный интернет).
* **Adapter 2:** `VMnet2` (Кабель в сторону HQ-RTR).
* **Adapter 3:** `VMnet3` (Кабель в сторону BR-RTR).

### 🏢 Центральный офис (HQ)
* **HQ-RTR (Роутер):** * Adapter 1: `VMnet2` (Смотрит на ISP).
  * Adapter 2: `VMnet4` (Смотрит в локальную сеть LAN).
* **HQ-SRV (Сервер):** Adapter 1: `VMnet4`.
* **HQ-CLI (Клиент):** Adapter 1: `VMnet4`. *(Временно можно добавить адаптер NAT для скачивания пакетов, но его **ОБЯЗАТЕЛЬНО** нужно отключить перед проверкой!)*

### 🏢 Филиал (BR)
* **BR-RTR (Роутер):**
  * Adapter 1: `VMnet3` (Смотрит на ISP).
  * Adapter 2: `VMnet5` (Смотрит в локальную сеть LAN).
* **BR-SRV (Сервер):** Adapter 1: `VMnet5`.

*(Номера `VMnet` могут быть любыми, главное — соблюдать парность соединений).*

## 🚀 Инструкция по запуску скрипта

### Шаг 1: Подготовка машин
1. Запустите все виртуальные машины.
2. Убедитесь, что на них **временно есть интернет** (через NAT-адаптеры) для скачивания пакетов (`frr`, `bind9`, `isc-dhcp-server`).
3. Зайдите на каждую машину под пользователем `root`:
   ```bash
   sudo su

Шаг 2: Создание скрипта
На каждой машине создайте файл, вставьте код скрипта и выдайте права на исполнение:
nano mod1.sh
# ... (вставляем код скрипта) ...
# Сохраняем: Ctrl+O -> Enter -> Ctrl+X

chmod +x mod1.sh

Шаг 3: Порядок запуска (КРИТИЧНО!)
Запускайте скрипт ./mod1.sh строго в следующем порядке, выбирая соответствующую роль в меню:
 * ISP — Запускаем первым, чтобы появился NAT-интернет для остальных машин.
 * HQ-RTR и BR-RTR — Настраивают туннели и OSPF. Подождите 10-15 секунд после их настройки, чтобы туннель поднялся.
 * HQ-SRV и BR-SRV — Серверы получают статические IP-адреса и настраивают DNS.
 * HQ-CLI — Запускаем в самом конце! Ему нужен работающий роутер HQ-RTR, чтобы получить IP-адрес по DHCP.
🔧 Шаг 4: Обязательные ручные действия ПОСЛЕ скрипта
Как только скрипт отработал на всех машинах, выполните эти действия, чтобы инфраструктура работала корректно:
 * Отключите временные NAT-адаптеры! * Зайдите в настройки VMware для машин HQ-CLI, HQ-SRV и BR-SRV и отключите адаптеры, которые давали временный интернет (снимите галочку Connected). Оставьте только локальные VMnet. Если этого не сделать, DNS-запросы будут уходить в Google (8.8.8.8), а не на наш сервер экзамена!
 * Перезагрузите сеть на HQ-CLI (или просто саму машину), чтобы она подтянула правильный DNS:
   sudo netplan apply

✅ Чек-лист проверки (Как у экспертов)
После выполнения всех шагов, зайдите на HQ-CLI и проверьте свою работу:
1. Проверка маршрутизации и OSPF
Зайдите на HQ-RTR и введите:
sudo vtysh -c "show ip ospf neighbor"

 * Ожидаемый результат: Наличие соседа в состоянии Full.
Зайдите на HQ-CLI и проверьте связь офисов:
ping 192.168.30.2 -c 4

 * Ожидаемый результат: Пинг идет успешно (связь между клиентом центра и сервером филиала установлена).
2. Проверка DNS (Прямая и Обратная зоны)
На машине HQ-CLI:
nslookup hq-rtr.au-team.irpo

 * Ожидаемый результат: Возвращает IP-адрес 192.168.10.1.
nslookup 192.168.10.1

 * Ожидаемый результат: Возвращает имя hq-rtr.au-team.irpo (Работает PTR-запись).
3. Проверка безопасности (SSH)
На машине HQ-CLI подключитесь к серверу:
ssh sshuser@192.168.10.2 -p 3015

 * Ожидаемый результат: Появится баннер "Authorized access only", система запросит пароль (вводим P@ssw0rd) и успешно авторизует. После входа команда sudo su должна работать без запроса пароля.
