# 🏆 Демоэкзамен 2025: Модуль 1 (Сетевая инфраструктура)

Автоматизированный скрипт для полного выполнения Модуля 1 (Настройка сети, VLAN, GRE, OSPF, DHCP, DNS, SSH) строго по заданию Демоэкзамена 2025 года.

🔗 **Скрипт:** [`iti2.1.sh`](https://github.com/IslomTI/demos/blob/main/iti2.1.sh)

---

## ⚠️ ВАЖНО: Подготовка виртуальной среды (VMware)
*90% ошибок на экзамене (не работает пинг, не поднимается OSPF) происходят из-за неправильно подключенных виртуальных проводов!*

Перед запуском скрипта убедитесь, что сетевые адаптеры (Network Adapters) виртуальных машин настроены следующим образом:

### 1. Планирование виртуальных коммутаторов (VMnet)
В `Virtual Network Editor` (или просто в настройках ВМ) распределите сети:
* **NAT (обычно VMnet8)** — Для временного интернета (скачивание пакетов).
* **VMnet2** — Линк `ISP` <---> `HQ-RTR`.
* **VMnet3** — Линк `ISP` <---> `BR-RTR`.
* **VMnet4 (LAN HQ)** — Локальная сеть Центра (HQ-RTR, HQ-SRV, HQ-CLI).
* **VMnet5 (LAN BR)** — Локальная сеть Филиала (BR-RTR, BR-SRV).

### 2. Подключение адаптеров к машинам
Настройте машины строго по этой схеме:
* **ISP:** 3 адаптера (1. `NAT`, 2. `VMnet2`, 3. `VMnet3`)
* **HQ-RTR:** 2 адаптера (1. `VMnet2`, 2. `VMnet4`)
* **BR-RTR:** 2 адаптера (1. `VMnet3`, 2. `VMnet5`)
* **HQ-SRV:** 1 адаптер (`VMnet4`)
* **BR-SRV:** 1 адаптер (`VMnet5`)
* **HQ-CLI:** 1 адаптер (`VMnet4`)

> **Лайфхак для экзамена:** Чтобы скрипт смог скачать нужные пакеты (frr, bind9), **на время запуска скрипта** добавьте всем машинам дополнительный сетевой адаптер `NAT`. После того как скрипт отработает, просто удалите его или снимите галочку "Connected" у всех, кроме ISP.

## 🚀 Пошаговая инструкция по запуску

### Шаг 1. Скачивание скрипта
Зайдите на каждую машину под пользователем `root` и скачайте скрипт с GitHub:

```bash
sudo su
wget [https://raw.githubusercontent.com/IslomTI/demos/main/iti2.1.sh](https://raw.githubusercontent.com/IslomTI/demos/main/iti2.1.sh)
chmod +x iti2.1.sh

Шаг 2. Запуск и настройка
Запускайте скрипт на машинах строго по очереди. Скрипт попросит указать имена интерфейсов (например, ens33, ens37). Посмотреть их можно прямо в меню скрипта (они будут выведены желтым цветом).
Выполните команду:
./iti2.1.sh

Порядок настройки (рекомендуемый):
 * Выберите 1) ISP. Укажите интерфейс с интернетом (NAT), затем интерфейс смотрящий на HQ, затем на BR.
 * Выберите 2) HQ-RTR. Укажите интерфейс к ISP, затем интерфейс смотрящий внутрь (в LAN).
 * Выберите 3) BR-RTR. Укажите интерфейс к ISP, затем в LAN.
 * Выберите 4) HQ-SRV. Укажите его единственный локальный интерфейс.
 * Выберите 5) BR-SRV. Укажите его локальный интерфейс.
 * Выберите 6) HQ-CLI. Настраиваем в последнюю очередь!
✅ Чек-лист проверки (Как проверить себя)
После того как все 6 машин настроены, сядьте за HQ-CLI (машину с графическим интерфейсом) и проверьте работу инфраструктуры:
1. Проверка выдачи адреса (DHCP)
ip a

Убедитесь, что машина получила IP-адрес из диапазона 192.168.20.X.
2. Проверка маршрутизации и GRE-туннеля
ping 192.168.30.2 -c 4

Пинг до сервера филиала должен быть успешным. Это доказывает работу NAT, OSPF и GRE-туннеля.
3. Проверка OSPF
Зайдите на HQ-RTR и выполните:
sudo vtysh -c "show ip ospf neighbor"

В списке должен быть сосед 2.2.2.2 в состоянии Full.
4. Проверка DNS (Задание 10)
На машине HQ-CLI введите:
nslookup hq-rtr.au-team.irpo   # Проверка прямой зоны (вернет 192.168.10.1)
nslookup 192.168.10.1          # Проверка обратной PTR-зоны (вернет hq-rtr...)
nslookup ya.ru                 # Проверка пересылки DNS в интернет

5. Проверка безопасности (Задание 5)
На машине HQ-CLI попробуйте подключиться к серверу:
ssh sshuser@192.168.10.2 -p 3015

Вас должно пустить по паролю P@ssw0rd, показать баннер Authorized access only, а команда sudo su внутри сервера должна работать без запроса пароля.
Удачи на экзамене! Инфраструктура полностью готова к сдаче.
