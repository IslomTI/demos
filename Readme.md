# 🏆 Демоэкзамен 2026 (Вариант 2): Полное руководство
Это подробная «шпаргалка» для безошибочного развертывания инфраструктуры демоэкзамена 2026 года (Базовый уровень). Скрипты делают **всю** грязную работу: настраивают ROAS, OSPF, RAID5, Samba AD, Ansible, Docker и DNS.
От тебя требуется только правильно склонировать виртуалки, воткнуть «провода» (LAN Segments) и запустить скрипты в правильном порядке.
🔗 **В репозитории два файла:**
 * iti1.sh — Поднимает всю сеть (Мосты, OSPF, DHCP, NTP, SSH-доступы).
 * iti2.sh — Устанавливает сервисы (RAID5, Samba AD, Docker, Apache, Ansible, Nginx).
## 🛠️ ЭТАП 1: Подготовка виртуальных машин (VMware)
На экзамене выдают всего два чистых образа: **Ubuntu Server** (без графики) и **Ubuntu Desktop** (с графикой для HQ-CLI). Твоя задача — сделать из них 7 машин.
### 1. Клонирование
В VMware нажми правой кнопкой на чистый образ ➔ Manage ➔ Clone... ➔ Create a full clone (Полный клон). Сделай так:
 * Из **Ubuntu Server** сделай 6 клонов: ISP, HQ-RTR, HQ-SW, BR-RTR, HQ-SRV, BR-SRV.
 * Из **Ubuntu Desktop** сделай 1 клон: HQ-CLI.
### 2. Настройка Оперативной памяти (RAM)
Зайди в настройки каждой ВМ (Edit virtual machine settings) и выставь память:
 * **BR-SRV** — **3 ГБ** (или 4 ГБ). Ему будет тяжело (там Docker и Контроллер домена).
 * **HQ-SRV** — **2 ГБ**.
 * **Все остальные** (ISP, HQ-RTR, HQ-SW, BR-RTR, HQ-CLI) — по **1 ГБ**.
### 3. Добавление дисков для RAID5 (Только на HQ-SRV)
 1. Открой настройки **HQ-SRV**.
 2. Нажми Add... ➔ Hard Disk ➔ SCSI или SATA ➔ Create a new virtual disk.
 3. Укажи размер **1 GB**.
 4. Повтори это действие еще 2 раза. В итоге у HQ-SRV должно быть 3 новых дополнительных диска по 1 ГБ каждый. Скрипт сам соберет из них RAID5.
## 🌐 ЭТАП 2: Подключение проводов (LAN Segments)
Мы не используем VMnet. Мы используем LAN Segments — это изолированные виртуальные провода.
В настройках любой виртуалки перейди в Network Adapter ➔ LAN Segments... ➔ Add и создай 6 сегментов:
ISP-HQ, ISP-BR, HQ-Net, SRV-Net, CLI-Net, BR-Net.
Теперь добавь адаптеры машинам строго по этой таблице (лишние удали кнопкой Remove, недостающие добавь кнопкой Add -> Network Adapter):
| Имя машины | Адаптер 1 (Интернет) | Адаптер 2 (LAN Segment) | Адаптер 3 (LAN Segment) | Адаптер 4 (LAN Segment) |
|---|---|---|---|---|
| 🌍 **ISP** | NAT | ISP-HQ | ISP-BR | - |
| 🖧 **HQ-RTR** | NAT | ISP-HQ | HQ-TRUNK | - |
| 🖧 **HQ-SW** | NAT | HQ-Net | SRV-Net | CLI-Net |
| 🖧 **BR-RTR** | NAT | ISP-BR | BR-Net | - |
| 🖥️ **HQ-SRV** | NAT | SRV-Net | - | - |
| 🖥️ **BR-SRV** | NAT | BR-Net | - | - |
| 💻 **HQ-CLI** | NAT | CLI-Net | - | - |
## 🚀 ЭТАП 3: Инструкция по запуску
### Шаг 1. Скачивание скриптов (Пока есть интернет)
Запусти все машины. Авторизуйся везде (на серверах введи sudo su, чтобы стать root).
На машинах с серверами пишем:
```bash
apt update && apt install git -y
git clone https://github.com/islomti/demos
cd demos
chmod +x iti1.sh iti2.sh

```
**Особенность HQ-CLI (Ubuntu Desktop):** На нем нет git по умолчанию. Открой терминал и напиши:
```bash
sudo su
apt update && apt install git -y
git clone https://github.com/IslomTI/demos
cd demos
chmod +x iti1.sh iti2.sh

```
### Шаг 2. ИЗОЛЯЦИЯ СЕТЕЙ (САМОЕ ВАЖНОЕ!) 🛑
Не выключая машины, перейди в их настройки в VMware (VM -> Settings).
У **ВСЕХ машин КРОМЕ ISP** выбери первый адаптер (Network Adapter 1, который NAT) и **СНИМИ галочку "Connected"** сверху справа. Мы имитируем отсутствие прямого интернета у локальных машин. У ISP галочку оставляем!
> 💡 *Когда скрипт будет просить ввести имя интерфейса (например, ens33 или ens34), смотри на вывод в терминале. Вводи те интерфейсы, у которых статус UP (они зеленые или активные). Отключенный NAT интерфейс вводить нельзя!*
> 
### Шаг 3. ЗАПУСК МОДУЛЯ 1 (Строим сеть)
Запускай скрипт ./iti1.sh строго по очереди. Жди, пока скрипт напишет "УСПЕХ", прежде чем переходить к следующей машине:
 1. **ISP** ➔ Пункт 1
 2. **HQ-RTR** ➔ Пункт 2
 3. **HQ-SW** ➔ Пункт 3
 4. **BR-RTR** ➔ Пункт 4
 5. **HQ-SRV** ➔ Пункт 5
 6. **BR-SRV** ➔ Пункт 6
 7. **HQ-CLI** ➔ Пункт 7 *(дождись получения IP)*
📸 *Сделай Snapshot (снимок) всех виртуальных машин в VMware! Фундамент залит.*
### Шаг 4. ЗАПУСК МОДУЛЯ 2 (Поднимаем сервисы)
Тут можно запускать скрипт ./iti2.sh параллельно (одновременно) на всех машинах, чтобы не терять время:
 * **ISP** ➔ Пункт 1 (Прокси и пароли)
 * **HQ-SRV** ➔ Пункт 5 (RAID5, Базы данных)
 * **BR-SRV** ➔ Пункт 6 (Контроллер домена, Ansible, Docker)
 * **HQ-CLI** ➔ Пункт 7 (Ввод в домен)
🔄 **ФИНАЛ:** Обязательно перезагрузи HQ-CLI (команда reboot), чтобы применились групповые политики.
## 📋 Команды для отчета экспертам (Чек-лист)
Во время экзамена тебе нужно делать скриншоты для отчета. Вставляй эти команды в терминал, делай скриншоты вывода и вставляй в Word-файл.
### 🛡️ Модуль 1: Маршрутизация и Сеть
**1. Проверка выдачи адресов (HQ-CLI):**
```bash
ip a

```
*(Должен быть адрес 192.168.20.10 или выше).*
**2. Проверка туннеля и динамической маршрутизации (HQ-RTR):**
```bash
vtysh -c "show ip ospf neighbor"
vtysh -c "show ip route"

```
*(Должен быть виден сосед 2.2.2.2 (Full) и маршруты с буквой O (OSPF)).*
**3. Проверка удаленного доступа SSH и DNAT (HQ-CLI):**
*(В терминале обычного пользователя, не root)*
```bash
ssh sshuser@192.168.10.2 -p 2012
# Пароль: P@ssw0rd (Должен появиться баннер Authorized access only)

```
**4. Проверка NTP (ISP и клиенты):**
На ISP проверяем Stratum:
```bash
chronyc tracking

```
На BR-SRV проверяем синхронизацию:
```bash
chronyc sources

```
### ⚙️ Модуль 2: Сервисы и Автоматизация
**1. Проверка аппаратного RAID-5 (HQ-SRV):**
```bash
mdadm --detail /dev/md2
lsblk

```
*(Должен быть State: clean, 3 working devices).*
**2. Проверка контроллера домена Samba AD (BR-SRV):**
```bash
samba-tool domain info AU-TEAM.IRPO
samba-tool user list

```
*(Должны быть пользователи hquser1...hquser5).*
**3. Автоматизация Ansible (BR-SRV):**
```bash
ansible all -m ping

```
*(Все узлы из файла инвентаризации должны ответить зеленым SUCCESS / pong).*
**4. Проверка AutoFS и SSSD (HQ-CLI):**
На экране блокировки Ubuntu зайди под пользователем hquser1 с паролем P@ssw0rd123!. В терминале напиши:
```bash
id
df -h | grep nfs

```
*(Папка /raid должна быть примонтирована автоматически).*
**5. Проверка Web-сервисов (HQ-CLI):**
Открой установленный Яндекс Браузер и введи:
 1. http://web.au-team.irpo ➔ Попросит пароль. Введи Kozmac / P@ssw0rd. Откроется сайт на Apache из HQ-SRV.
 2. http://docker.au-team.irpo ➔ Откроется заглушка сайта из Docker-контейнера на BR-SRV.
