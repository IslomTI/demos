# 🏆 Демоэкзамен 2025: Автоматизация Модулей 1 и 2

![Bash](https://img.shields.io/badge/Language-Bash-4EAA25?style=for-the-badge&logo=gnu-bash&logoColor=white)
![Ubuntu](https://img.shields.io/badge/OS-Ubuntu_Linux-E95420?style=for-the-badge&logo=ubuntu&logoColor=white)

Автоматизированный shell-скрипт для полного и безошибочного развертывания инфраструктуры Демоэкзамена 2025 года (Сетевое и системное администрирование). 

Скрипт реализует настройку:
**Сети и маршрутизации:** VLAN, GRE-туннели, OSPF, DHCP, NAT, Port Forwarding.
**Инфраструктурных сервисов:** DNS (Bind9), NTP (Chrony), SSH.
**Прикладных сервисов:** Nginx Reverse Proxy, аппаратный RAID0, NFS (AutoFS), Samba AD DC, Docker (MediaWiki), Apache (Moodle).

🔗 **Скрипт:** [`5.sh`](https://github.com/IslomTI/demos/blob/main/5.sh)

## ⚠️ ЭТАП 1: Аппаратная подготовка ВМ (КРИТИЧНО)

Если не выполнить эти требования **до** запуска виртуальных машин, установка сервисов зависнет, или скрипт выдаст ошибку при сборке RAID-массива.

### 1. Распределение ОЗУ (RAM)
* **`BR-SRV`** — Минимум **3 ГБ** (рекомендуется 4 ГБ). Это самый нагруженный узел (контроллер домена Samba AD DC + контейнеры Docker).
* **`HQ-SRV`** — Минимум **2 ГБ**. (Обслуживает базы данных MariaDB и веб-сервер Moodle).
* **`ISP`, `HQ-RTR`, `BR-RTR`, `HQ-CLI`** — Достаточно по **1 ГБ**.

### 2. Подготовка дисков для RAID0 (Только для HQ-SRV)
В настройках виртуальной машины **HQ-SRV** необходимо вручную добавить жесткие диски:
1. Нажмите `Add...` ➔ `Hard Disk`.
2. Выберите тип **SCSI** или **SATA** (Обязательно!).
3. Добавьте **ТРИ новых пустых диска** (например, по 5 ГБ каждый).
> 💡 *Скрипт автоматически просканирует шину, найдет эти 3 пустых диска и соберет из них аппаратный RAID0 массив `/dev/md0` для расшаривания по NFS.*

## 🌐 ЭТАП 2: Топология и Сетевые адаптеры (VMware)

В `Virtual Network Editor` создайте 4 изолированные сети (тип *Host-only*, снимите галочку локального DHCP):
* **VMnet10** — Магистраль `ISP` ⟷ `HQ-RTR`
* **VMnet11** — Магистраль `ISP` ⟷ `BR-RTR`
* **VMnet12** — Локальная сеть `HQ-Net` (Центральный офис)
* **VMnet13** — Локальная сеть `BR-Net` (Филиал)

### Схема подключения адаптеров в настройках ВМ:

| Имя машины | Адаптер 1 (Временный) | Адаптер 2 | Адаптер 3 |
| :--- | :--- | :--- | :--- |
| 🌍 **ISP** | `NAT` (Интернет) | `VMnet10` (к HQ) | `VMnet11` (к BR) |
| 🖧 **HQ-RTR** | `NAT` | `VMnet10` (к ISP) | `VMnet12` (LAN HQ) |
| 🖧 **BR-RTR** | `NAT` | `VMnet11` (к ISP) | `VMnet13` (LAN BR) |
| 🖥️ **HQ-SRV** | `NAT` | `VMnet12` (LAN HQ) | - |
| 🖥️ **BR-SRV** | `NAT` | `VMnet13` (LAN BR) | - |
| 💻 **HQ-CLI** | `NAT` | `VMnet12` (LAN HQ) | - |

## 🚀 ЭТАП 3: Инструкция по запуску (Каскадный метод)

Механика работы скрипта полагается на строгий порядок запуска. Интернет передается вглубь сети последовательно.

### Шаг 1. Скачивание скрипта
Включите все машины. Убедитесь, что интернет работает (через первый `NAT` адаптер). Зайдите на каждую ВМ под пользователем `root` (`sudo su`) и выполните:
```bash
wget [https://raw.githubusercontent.com/IslomTI/demos/main/5.sh](https://raw.githubusercontent.com/IslomTI/demos/main/5.sh)
chmod +x 5.sh

```

### Шаг 2. ИЗОЛЯЦИЯ СЕТЕЙ (Самый важный шаг!)

Не выключая машины, перейдите в их настройки в VMware (`VM -> Settings`).
У **ВСЕХ машин КРОМЕ ISP** выберите первый адаптер (`NAT`) и **СНИМИТЕ галочку "Connected"**.

> 🛑 **Внимание:** Не удаляйте адаптер кнопкой Remove, просто отключите его галочкой. Этим мы отключаем прямой интернет, заставляя трафик идти через наши настроенные роутеры.

### Шаг 3. Запуск по цепочке

Запускайте `./5.sh` строго в указанном порядке. Скрипт попросит ввести имена интерфейсов (например, `ens33`).

1. Сначала: **ISP** (В меню скрипта выберите пункт `1`).
2. Затем роутеры: **HQ-RTR** (пункт `2`) и **BR-RTR** (пункт `3`).
3. Затем серверы: **HQ-SRV** (пункт `4`) и **BR-SRV** (пункт `5`).
4. В самом конце: **HQ-CLI** (пункт `6`).
* *После настройки клиента обязательно выполните команду `reboot`, чтобы применились групповые политики домена.*

## 📸 Чек-лист для отчета экспертам

После завершения настройки пройдитесь по машинам и сделайте скриншоты выполнения этих команд для подтверждения работоспособности инфраструктуры.

### 🛡️ Модуль 1: Сетевая инфраструктура

**1. Проверка выдачи адресов (DHCP)**
На клиенте **HQ-CLI** выполните:

```bash
ip a

```

*Ожидаемый результат: Наличие IP-адреса из пула 192.168.20.10-30.*

**2. Проверка динамической маршрутизации и GRE**
На роутере **HQ-RTR** выполните:

```bash
vtysh -c "show ip ospf neighbor"

```

*Ожидаемый результат: Виден сосед `2.2.2.2` в состоянии `Full`.*

С клиента **HQ-CLI** пропингуйте сервер филиала:

```bash
ping 192.168.30.2 -c 4

```

**3. Проверка DNS (Прямая, обратная зона и пересылка)**
На клиенте **HQ-CLI** выполните:

```bash
nslookup hq-rtr.au-team.irpo
nslookup 192.168.10.1
nslookup ya.ru

```

**4. Проверка безопасного удаленного доступа**
На клиенте **HQ-CLI** подключитесь к серверу по нестандартному порту:

```bash
ssh sshuser@192.168.10.2 -p 3015

```

*Ожидаемый результат: Появление баннера "Authorized access only". Пароль для входа: `P@ssw0rd`.*

### ⚙️ Модуль 2: Сервисы

**1. Проверка аппаратного RAID0**
На сервере **HQ-SRV** выполните:

```bash
lsblk
mdadm --detail /dev/md0

```

*Ожидаемый результат: Видно 3 диска, объединенных в активный массив `md0`.*

**2. Проверка контроллера домена Samba AD DC**
На сервере **BR-SRV** выполните:

```bash
samba-tool domain info 127.0.0.1
samba-tool user list

```

*Ожидаемый результат: Информация о домене AU-TEAM.IRPO и список созданных пользователей (user1..5.hq).*

**3. Проверка синхронизации времени (NTP)**
На сервере **HQ-SRV** или **BR-SRV** выполните:

```bash
chronyc sources

```

*Ожидаемый результат: Звездочка `*` слева от IP-адреса роутера, доказывающая успешную синхронизацию.*

**4. Проверка AutoFS и входа в домен**
На клиенте **HQ-CLI** (в графическом интерфейсе войдите под `Administrator@au-team.irpo` с паролем `P@ssw0rd123!`). В терминале:

```bash
id
df -h | grep nfs

```

*Ожидаемый результат: Команда `id` покажет доменные группы, а `df -h` — автоматически примонтированную по NFS папку RAID-массива.*

**5. Проверка Web-сервисов (Nginx Reverse Proxy + Docker)**
В графическом интерфейсе **HQ-CLI** откройте Яндекс Браузер и сделайте скриншоты двух страниц:

1. Адрес `http://moodle.au-team.irpo` *(Откроется Moodle, работающий на Apache HQ-SRV).*
2. Адрес `http://wiki.au-team.irpo` *(Откроется MediaWiki, работающая в Docker-контейнере на BR-SRV).*
